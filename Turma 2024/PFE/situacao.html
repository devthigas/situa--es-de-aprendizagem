2 - Estratégia de aprendizagem desafiadora

Situação-problema ( X )   Estudo de caso (  )  Pesquisa Aplicada (   )  Projeto (  ) Integrador (  )

Contextualização

Você foi contratado para desenvolver, em colaboração com a equipe de UX Design, um painel administrativo flexível e comercializável, projetado para atender a diferentes necessidades. Esse painel servirá como base para diversas aplicações, permitindo que os compradores personalizem e integrem facilmente em seus projetos.

Para garantir que a solução atenda às expectativas do usuário final, a equipe de UX realizou uma pesquisa detalhada, identificando requisitos essenciais e criando um protótipo de baixa fidelidade, que servirá como referência para o design e a experiência do painel.

Para otimizar o desenvolvimento e facilitar futuras customizações, a equipe definiu o uso do Tailwind CSS, um framework que oferece agilidade, flexibilidade e alta capacidade de personalização.

Objetivo

Desenvolver um painel administrativo responsivo, utilizando Tailwind CSS, que seja intuitivo, esteticamente agradável e de fácil adaptação para diferentes cenários comerciais. O painel deverá seguir as melhores práticas de UI e UX Design, garantindo uma experiência interativa e acessível.

Desafio

Criar um painel administrativo genérico, que possa ser utilizado em diferentes contextos, garantindo uma interface interativa, responsiva e modular. Além disso, o código deve ser estruturado de forma organizada, permitindo facilidade de personalização para diferentes clientes.

  • Implementação do protótipo de alta fidelidade, garantindo a fidelidade ao design e a experiência do usuário na versão desktop e mobile.
  • Modais interativos, para exibição de informações ou confirmação de ações.
  • Menus dinâmicos, adaptáveis a diferentes tamanhos de tela, proporcionando uma navegação fluida.
  • Animações sutis, para aprimorar a experiência do usuário sem comprometer o desempenho.

Entregas

  • ✅ Protótipo de alta fidelidade: Criado no Figma (enviar o link).
  • ✅ Código documentado: Enviar o código-fonte compactado (ZIP).
  • ✅ Publicação no GitHub Pages: Disponibilizar o projeto para acesso público.
